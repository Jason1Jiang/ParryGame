# 快速开始指南 v3.5 - 云端排行榜

## 🎮 游戏功能

### 排行榜系统
v3.5 版本新增了全球云端排行榜功能，玩家可以：
- 提交自己的游戏分数
- 查看全球玩家排名
- 按难度查看不同排行榜
- 离线查看缓存数据

## 🚀 快速测试

### 1. 提交分数
```
1. 开始游戏（选择任意难度）
2. 游戏结束后等待2秒
3. 输入你的名字（最多20字符）
4. 点击"提交"按钮
5. 分数自动上传到排行榜
```

### 2. 查看排行榜
```
1. 在主菜单点击 "RANKS" 按钮
2. 默认显示硬核模式排行榜
3. 点击难度标签切换：
   - 硬核 (Hardcore)
   - 平衡 (Balanced)
   - 休闲 (Casual)
4. 点击"返回"回到主菜单
```

### 3. 跳过提交
```
如果不想提交分数：
1. 游戏结束后出现名字输入界面
2. 点击"跳过"按钮
3. 直接返回结算界面
```

## 📊 排行榜说明

### 排名规则
1. **主要排序**: 击杀数（从高到低）
2. **次要排序**: 存活时间（从高到低）
3. **显示数量**: 前100名

### 奖牌系统
- 🥇 第1名：金牌
- 🥈 第2名：银牌
- 🥉 第3名：铜牌

### 难度分类
每个难度有独立的排行榜：
- **硬核模式**: 极快节奏，15敌人上限
- **平衡模式**: 快速节奏，12敌人上限
- **休闲模式**: 适中节奏，8敌人上限

## 🔧 技术说明

### Firebase 配置
项目使用 Firebase Realtime Database 存储排行榜数据：
- 项目ID: `bladeecho-d4cd3`
- 数据库: `https://bladeecho-d4cd3-default-rtdb.firebaseio.com`

### 离线支持
- 排行榜数据会自动缓存到本地
- 离线时显示缓存的数据
- 网络恢复后自动同步最新数据

### 实时更新
- 排行榜数据实时同步
- 其他玩家提交分数后自动更新
- 无需刷新页面

## 🎯 使用场景

### 场景1: 竞技挑战
```
1. 查看当前排行榜
2. 记住第一名的分数
3. 开始游戏挑战
4. 提交分数查看排名
```

### 场景2: 难度对比
```
1. 在不同难度下游戏
2. 提交分数到各难度排行榜
3. 对比自己在不同难度的表现
```

### 场景3: 离线游戏
```
1. 在线时查看一次排行榜（缓存数据）
2. 断网后仍可查看缓存的排行榜
3. 恢复网络后数据自动更新
```

## ⚠️ 注意事项

### 名字输入
- 最多20个字符
- 不能为空
- 支持中文、英文、数字

### 网络要求
- 首次使用需要网络连接
- 提交分数需要网络连接
- 查看排行榜可离线（使用缓存）

### 浏览器兼容
- 推荐使用现代浏览器
- Chrome 90+
- Firefox 88+
- Safari 14+
- Edge 90+

## 🐛 常见问题

### Q: 提交分数失败怎么办？
A: 检查网络连接，确保可以访问 Firebase 服务。

### Q: 排行榜不更新？
A: 刷新页面或检查网络连接。

### Q: 离线能提交分数吗？
A: 不能，提交分数需要网络连接。但可以查看缓存的排行榜。

### Q: 可以修改已提交的分数吗？
A: 不能，每次提交都是新记录。

### Q: 排行榜最多显示多少名？
A: 显示前100名玩家。
